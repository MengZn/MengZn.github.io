CRI-O

LIGHTWEIGHT CONTAINER RUNTIME FOR KUBERNETES

CRI-O是實作了Kubernetes CRI（Container Runtime Interface)，可以使用任何符合OCI(Open Container Initiative)的runtime spec 作為Kubernetes pod的容器運執行平台，目前CRI-O支援runc和Clear Containers。

Kubeadm

kubeadm是 Kubernetes 官方推出的部署工具，只要調整一些初始化參數就可以啟動許多Alpha、Beta功能，可以看Kubeadm Roadmap來了解未來的功能發展。

環境配置

本次安裝是使用三台虛擬機器去作部署，作業系統採用的是Ubuntu 16.04 LTS版。

部署Kubeadm

『kubeadm版本是Kubernetes1.12』，可以參考官方網站的安裝方式。

本篇的指令皆需要sudo權限

apt-get update && apt-get install -y apt-transport-https curl
curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | apt-key add -
cat <<EOF >/etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF
apt-get update
apt-get install -y kubelet kubeadm kubectl
apt-mark hold kubelet kubeadm kubectl

部署CRI-O

『CRI-O 版本採用的是1.11』，可以參考官方網站的安裝方式。

這邊需要開啟Linux Kernel內的overlay、br_netfilter、ip_forward model

modprobe overlay
modprobe br_netfilter

cat > /etc/sysctl.d/99-kubernetes-cri.conf <<EOF
net.bridge.bridge-nf-call-iptables  = 1
net.ipv4.ip_forward                 = 1
net.bridge.bridge-nf-call-ip6tables = 1
EOF

sysctl --system

如果想從source code build 的朋友，未來會有一篇在介紹CRI-O的介紹與操作。

透過ppa安裝CRI-O 

apt-get update
apt-get install software-properties-common -y

add-apt-repository ppa:projectatomic/ppa -y
apt-get update
apt-get install cri-o-1.11 -y

透過 systemd 啟動crio backend server 

systemctl start crio

可以透過以下指令確認CRI-O是否成功安裝 

$crictl ps
CONTAINER ID        IMAGE        CREATED             STATE               NAME                      ATTEMPT             POD ID

修改kubelet

這邊需要修改kubelet的啟動參數cgroup-driver，因為kubelet預設的 cgroup-driver 啟動參數為 cgroupfs，但是CRI-O的 cgroup-driver 預設採用的是 systemd。

未來將會有一篇介紹 cgroup-driver 內 systemd 與 cgroupfs 不同的地方。

cat <<EOF > /etc/default/kubelet
KUBELET_EXTRA_ARGS=--cgroup-driver=systemd
EOF
systemctl daemon-reload
systemctl restart kubelet
