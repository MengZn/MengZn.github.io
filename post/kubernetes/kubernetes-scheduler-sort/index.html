Sort stage

由於scheduler是按照 FIFO 的順序調度Pod的

因此當 scheduler queue 裡出現多個等待調度的 Pod 時，可以對這些在 scheduler queue 裡的 Pod 的進行排序，把我們優先權較高 Pod 進行排序，讓 Pod 可以更優先的被調度。

目前在 scheduler source code 裡面只有一個 PrioritySort Sort 排序。

想要實踐一個 sort plugins需要實作以下幾個 interface。

// Plugin is the parent type for all the scheduling framework plugins.
type Plugin interface {
	Name() string
}

type QueueSortPlugin interface {
	Plugin
	// Less are used to sort pods in the scheduling queue.
	Less(*QueuedPodInfo, *QueuedPodInfo) bool
}

完成了這兩個 interface 就完成了一個 sort plugins ，看一下scheduler 裡面實作的PrioritySort。

// Name is the name of the plugin used in the plugin registry and configurations.
const Name = "PrioritySort"

// PrioritySort is a plugin that implements Priority based sorting.
type PrioritySort struct{}

var _ framework.QueueSortPlugin = &PrioritySort{}

// Name returns name of the plugin.
func (pl *PrioritySort) Name() string {
	return Name
}

//GetPodPriority會拿到pod spec裡面的Priority欄位
//當這個數值越大越容易會先被挑選出來
//另外當兩個pod Priority欄位數值一樣的時候
//會比較兩個pod被建立的時間，越早被建立的會先被挑選出來
func (pl *PrioritySort) Less(pInfo1, pInfo2 *framework.QueuedPodInfo) bool {
	p1 := pod.GetPodPriority(pInfo1.Pod)
	p2 := pod.GetPodPriority(pInfo2.Pod)
	return (p1 > p2) || (p1 == p2 && pInfo1.Timestamp.Before(pInfo2.Timestamp))
}

// New initializes a new plugin and returns it.
func New(_ runtime.Object, handle framework.FrameworkHandle) (framework.Plugin, error) {
	return &PrioritySort{}, nil
}
