minikube 在掛載本機遇到的神奇問題

2018-07-02
kubernetes
Page content

開發人員通常會使用minkube開發環境，紀錄當時遇到的掛載問題。

以下是我當時遇到狀況的環境，到V0.28.0還有此問題存在。

Note:

重現當時問題採用的是V0.28.0版本的minikube

Environment
minikube version v0.24.0
OS macOS
Driver virtualbox
ISO version minikube-v0.23.6.iso

我們要掛載資料夾或檔案至minkube的Virtual Machine 上的時候，會執行mount的動作。 可參考官方網站的掛載方式。

$minikube mount ~/app:/app
Mounting /home/jason/app into /app on the minikubeVM
This daemon process needs to stay alive for the mount to still be accessible...
ufs starting

透過minikube的指令連進虛擬機內

$minkube ssh

開發人員會將自己開發的程式透過Persistent Volumes的方式讓pod存取。 例如: 網頁程式設計人員撰寫PHP的程式碼，只要更新本地的程式碼minikube的pod就可以線上迭代，大幅地增加開發人員的速度。

我們要將app這個資料夾透過hostpath的方式掛載到minikube上

$cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: test-volume
spec:
  containers:
  - image: nginx
    name: test-volume
    volumeMounts:
    - mountPath: /app
      name: test-volume
  volumes:
  - name: test-volume
    hostPath:
      path: /app
EOF

透過kubectl 指令確認pod正常運作

$kubectl get pod
NAME                            READY     STATUS        RESTARTS   AGE
test-volume                     1/1       Running       0          1m

這實在pod內出現神奇的問題，透過kubectl連進pod去看一下。

$kubectl exec -ti test-volume sh

$ls -al
drwxr-xr-x   1 root root    4096 Jul  2 19:13 .
drwxr-xr-x   1 root root    4096 Jul  2 19:13 ..
-rwxr-xr-x   1 root root       0 Jul  2 19:13 .dockerenv
drwxrwxr-x   1  501 20        64 Jul  2 18:50 app
...

因為uid、gid的這個問題會導致無法存取app內的文件
