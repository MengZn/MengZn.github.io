簡單介紹一下golang的指標

Page content
  • 讓我頭痛很久的struct的指摽

學習golang也一段時間了，紀錄一下當時學習遇到的問題。

golang的指標

package main

import "fmt"

func main() {
	a := 1
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
	b := a
	fmt.Printf("b的記憶體位置%p \n", &b)
	fmt.Printf("b的記憶位址代表的值%v \n", b)
}

會print出

a的記憶體位置 0xc420016070
a的記憶位址代表的值 1
b的記憶體位置 0xc420016080
b的記憶位址代表的值 1

從value來看，a把值copy給b。

從記憶體來看，系統會要一塊記憶體區塊（b）放置至從a拷貝過來的value。

讓我頭痛很久的struct的指摽

package main

import (
	"fmt"
	"strings"
)
type sample struct {
	Name string
}
func changeName(para sample){
	fmt.Printf("傳入changeName function para的記憶體位置%p \n", &para)
	para.Name=strings.ToUpper(para.Name)
}
func main() {
	a := sample{Name: "this is a"}
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
	changeName(a)
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
}

輸出的結果

a的記憶體位置0xc42000e1e0
a的記憶位址代表的值 {this is a}
傳入changeName function para的記憶體位置0xc42000e200
a的記憶體位置0xc42000e1e0
a的記憶位址代表的值 {this is a}

為什麼沒有change name呢？

原因如下：

  1. 系統把a的value(sample{Name: “this is a”}) copy一份丟進changeName的方法之中。

  2. 這時候changeName所修改的value為a copy過去後的value，並不是原記憶體a位置所代表value。

如果我們改成這樣做就會成功修改value了

package main

import (
	"fmt"
	"strings"
)
type sample struct {
	Name string
}
func changeName(para sample)sample{
	fmt.Printf("傳入changeName function para的記憶體位置%p \n", &para)
	para.Name = strings.ToUpper(para.Name)
	return para
}
func main() {
	a := sample{Name: "this is a"}
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
	a = changeName(a)
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
}

這時候輸出得結果就會改變了

a的記憶體位置0x1040c128
a的記憶位址代表的值 {this is a}
傳入changeName function para的記憶體位置0x1040c140
a的記憶體位置0x1040c128
a的記憶位址代表的值 {THIS IS A}

這時可以成功修改value的原因為

changeName的方法回傳一個vale，讓a去接收該方法回傳的value。

另外一種方法

使用pointer的方法當作傳輸的方式，就可以對同一塊記憶體區域進行操作。

package main

import (
	"fmt"
	"strings"
)
type sample struct {
	Name string
}
func changeName(para *sample){
	fmt.Printf("傳入changeName function para的記憶體位置%p \n", para)
	para.Name = strings.ToUpper(para.Name)
}
func main() {
	a := sample{Name: "this is a"}
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
	changeName(&a)
	fmt.Printf("a的記憶體位置%p \n", &a)
	fmt.Printf("a的記憶位址代表的值 %v \n", a)
}

這時候輸出的結果

a的記憶體位置0xc4200881c0
a的記憶位址代表的值 {this is a}
傳入changeName function para的記憶體位置0xc4200881c0
a的記憶體位置0xc4200881c0
a的記憶位址代表的值 {THIS IS A}

使用a的記憶體位置傳入changeName方法中，該方法就可以直接對傳入的記憶體區塊進行操作！
